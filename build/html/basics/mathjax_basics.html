数学公式¶

\(\LaTeX\) 简介¶

Donald Knuth 在70年代设计 \(\TeX\) 开源排版系统的一大初衷就是为了在自己即将出版的《计算机程序设计艺术》上轻松实现专业的数学公式呈现. 其后继者 Leslie Lamport 80年代在 \(\TeX\) 的基础上实现推出了 \(\LaTeX\)，其实质是基于 \(\TeX\) 的各种功能、样式的宏包集，其实现了内容与呈现分离. 这让 \(\LaTeX\) 在学术界大受欢迎，因为它让科学工作者只需要几分钟学习时间就可以上手排版自己的学术论文. 发展至今，\(\TeX\) 用户群体已经十分壮大， TeX Users Group(TUG) 已经整合了所有 \(\TeX\) 相关资源，建立了 Comprehensive TeX Archive Network (CTAN) 宏包库，并同时维护着 TeX Live , MacTeX 和 proTeXt 三个发行版本.

注解

与 Word 的 .doc 等采用富文本格式不同，$TeX$所采用的是纯文本的文件格式，所以它不依赖特定的文本编辑软件，你完全可以用记事本来写 \(\TeX\)，甚至你都不需要鼠标.

随着因特网的发展，人们又把关注的焦点转向如何轻松实现数学公式在网页上的呈现，于是涌现了一大批如 MathJax、KaTeX 这些支持在网页支持 \(\TeX\) 方式呈现数学公式的库.

数学公式的处理对于橘子数学的海量数学试题库来说无疑是至关重要，于是网站采用$LaTeX$排版系统并自主研发了一个试卷宏包以实现了数学试卷的标准排版( 见 BHCexam 宏包 )，我们也就需要以 \(\TeX\) 的方式来录入数学公式. 下面我们将尽可能简单地介绍如何用 \(\TeX\) 表示数学公式，以求读者能够在10分钟左右的时间里掌握一些常见的用法可以在橘子数学网站上录入一些公式.

参见

阅读中译版教程 《110分钟了解LaTeX》 可以帮助你对 \(\LaTeX\) 有一个更全面的了解.

两种数学模式¶

inline mode

将数学公式夹在两个``$``之间, 使数学公式与普通文字混排在段落中.

Code：

$(a+b)^2=a^2+2ab+b^2$是一条 inline 模式的数学公式.

Output：

\((a+b)^2=a^2+2ab+b^2\) 是一条 inline 模式的数学公式.

display mode

将数学公式夹在 \[ 与 \] 之间,使数学公式单独放在一个段落并居中.

Code：

这是一条 display 模式的数学公式:\[ (a+b)^2=a^2+2ab+b^2 \]

Output：

这是一条 display 模式的数学公式:

\[(a+b)^2=a^2+2ab+b^2\]

警告

使用 display 模式注意控制公式的长度，以防超出屏幕或纸张.

换行与空格¶

在 \(\LaTeX\) 中：

  • 单独一次回车不换行算一次空格，两次及两次以上回车换一行;

  • 连续多个空格、制表符、单独一次回车算一次空格;

  • 跟在命令后面的空格会被忽略;

  • 强制空格可用 ~ 或 \quad，后者为4个空格;

  • 强制换行可用 \\.

特殊字符与转义¶

以下字符在 \(\LaTeX\) 中需要进行转义:

# $ % ^ & _ { } ~

Code:

\[\# \quad \$ \quad \% \quad \text{^} \quad \&
\quad \_ \quad \{ \quad \} \quad \text{~} \]

Output:

\[\# \quad \$ \quad \% \quad \text{^} \quad \& \quad \_ \quad \{ \quad \} \quad \text{~}\]

上标与下标¶

{ele}^{upper}, {ele}_{lower}

在 \(\LaTeX\) 中用 ^ 和 _ 标明上下标。注意上下标的内容(子公式)一般需要用花括号包裹,否则上下标只对后面的一个符号起作用.

Code:

注意$x^{10}$不要写成$x^10$.

Output:

注意 \(x^{10}\) 不要写成 \(x^10\).

大中小括号¶

一般情况下，在数学公式中除了大括号 { } 需要转义外，( )[ ] 可以直接使用. 但一些情况下在括号前使用 \left 和 \right 命令自动调整括号的高度，可以使数学公式更美观. 注意 \left 和 \right 命令必须成对使用.

Code:

注意下面两者的区别.

\[
    \left\{\frac{1}{3}+\left[\left(2^4\right)^{\frac{1}{3}}+4\right]\right\}^2
\]

\[
    \{\frac{1}{3}+[(2^4)^{\frac{1}{3}}+4]\}^2
\]

Output:

注意下面两者的区别.

\[\left\{\frac{1}{3}+\left[\left(2^4\right)^{\frac{1}{3}}+4\right]\right\}^2\]
\[\{\frac{1}{3}+[(2^4)^{\frac{1}{3}}+4]\}^2\]

命令与环境¶

\command

斜杠 \ 在 \(\TeX\) 中是转义符，其后跟一些英文单词就成为一条 命令 .

Code:

这是一些$\TeX$命令:\[\TeX \quad \times \quad \pi \quad \triangle\]

Output:

这是一些 \(\TeX\) 命令:

\[\TeX \quad \times \quad \pi \quad \triangle\]
\command[optional parameter]{parameter1}{parameter2}

有些 命令 需要接受一个或多个参数, 我们把参数用大括号 { } 包起来按顺序跟在命令后面, 可选参数则写在命令与所有参数之间的 [] 里.

一个参数的命令: $\overrightarrow{AB}$、$\bar{z}$、$\sqrt{3}$;

多个参数的命令: $\frac{1}{2}$;

含有可选参数的命令: $\sqrt[3]{5}$.

Output:

一个参数的命令: \(\overrightarrow{AB}\) 、\(\bar{z}\) 、\(\sqrt{3}\) ;

多个参数的命令: \(\frac{1}{2}\) ;

含有可选参数的命令: \(\sqrt[3]{5}\) .

environment

环境是分别以 \begin 和 \end 两条命令为首尾的代码块，环境中可以嵌套环境. \begin 命令可以包含除了环境名以外的其他参数.

\begin{environment}
...
\end{environment}

\begin{environment}[optional parameter]{parameter1}
...
\end{environment}

Code:

分段函数:
\[f(x)=\begin{cases}
    1 & x\in \mathbf{Q}
    0 & x\notin \mathbf{Q}
\end{cases}\]

Output:

分段函数:

\[\begin{split}f(x)=\begin{cases} 1 & x\in \mathbf{Q}\\ 0 & x\notin \mathbf{Q} \end{cases}\end{split}\]

字体说明¶

\mathrm

阿拉伯数字、运算符、函数名、常量(英文字母)用正体，若有专属命令则调用该命令，可以用 \mathrm 设置为正体;

Code:

注意 $sin$ 和 $\sin$ 以及 $3+2i$ 与 $3+2\mathrm{i}$ 的区别.

Output:

注意 \(sin\) 和 \(\sin\) 以及 \(3+2i\) 与 \(3+2\mathrm{i}\) 的区别.

\mathit

变量名、常量(希腊字母)用斜体，可以用 \mathit 设置为斜体(一般情况下保持默认即可);

Code:

注意$\pi \approx 3.1415926 \cdots$ 与 $\mathit{\pi \approx 3.1415926 \cdots}$ 的区别

Output:

注意 \(\pi \approx 3.1415926 \cdots\) 与 \(\mathit{\pi \approx 3.1415926 \cdots}\) 的区别

\mathbf

常用集合名用粗体，用 \mathbf 设置为粗体;

Code:

注意 $x \in R$ 与 $x in \mathbf{R}$ 的区别

Output:

注意 \(x \in R\) 与 \(x in \mathbf{R}\) 的区别
